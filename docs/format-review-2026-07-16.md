## 🚨 已修复的严重问题

### 1. ASS/SSA 解析器注释行处理错误

**问题描述**:
- **严重程度**: 🔴 高
- **位置**: [src/ass.rs:8-10](file:///Users/mankong/volumes/code/subtitle-rs/subtitler/src/ass.rs#L8-L10), [src/ass.rs:71-105](file:///Users/mankong/volumes/code/subtitle-rs/subtitler/src/ass.rs#L71-L105)
- **发现时间**: 发布前审查

**根本原因**:
1. 正则表达式只匹配以 `Dialogue:` 开头的行，无法识别以 `Comment:` 开头的注释行
2. `is_comment` 字段判断逻辑错误，无法正确识别注释行

**影响**:
- ASS/SSA 文件中的注释行会被完全忽略，导致数据丢失
- Effect 字段包含 "Comment" 的行无法正确标记为注释

**修复方案**:
```rust
// 修复前（第9行）
Regex::new(r"^Dialogue:\s*(?:\d+,)?(\d+):(\d+):(\d+)[,.](\d+),...")

// 修复后
Regex::new(r"^(?:Dialogue|Comment):\s*(?:\d+,)?(\d+):(\d+):(\d+)[,.](\d+),...")

// 修复前（第93行）
let is_comment = caps.get(14).is_some_and(|m| m.as_str().contains("Comment"));

// 修复后（第97-98行）
let is_comment = line.trim().to_lowercase().starts_with("comment:")
  || caps.get(14).is_some_and(|m| m.as_str().contains("Comment"));
```

**验证结果**: ✅ 所有测试通过，包括 `test_is_comment_uses_effect`

### 2. TTML 时间戳格式错误

**问题描述**:
- **严重程度**: 🔴 高
- **位置**: [src/ttml.rs:232-238](file:///Users/mankong/volumes/code/subtitle-rs/subtitler/src/ttml.rs#L232-L238)
- **发现时间**: 发布前审查

**根本原因**:
代码将时间戳分隔符从 `.` 错误地替换为 `,`，不符合 TTML 标准

**影响**:
- 生成的 TTML 文件使用错误的时间戳格式
- 可能导致某些播放器无法正确识别时间戳

**修复方案**:
```rust
// 修复前（第236-237行）
let start_ttml = start.replace('.', ",");
let end_ttml = end.replace('.', ",");

// 修复后（第233-238行）
let start = crate::utils::format_timestamp(sub.start, "WebVTT");
let end = crate::utils::format_timestamp(sub.end, "WebVTT");
// TTML uses '.' separator (same as WebVTT), no conversion needed
```

**验证结果**: ✅ 所有测试通过，时间戳格式正确
