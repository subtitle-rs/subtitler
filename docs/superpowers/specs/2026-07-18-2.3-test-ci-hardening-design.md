## 2. 技术方案

### 2.1 W1: WASM 测试（src/wasm.rs 内联）

```rust
// src/wasm.rs — 末尾加
#[cfg(test)]
#[wasm_bindgen_test]
fn test_parse_subtitles_srt() {
  let srt = "1\n00:00:01,000 --> 00:00:03,500\nHello\n\n";
  let result = parse_subtitles(srt);
  assert!(result.is_ok());
  assert_eq!(result.subtitle_count, 1);
  assert!(result.output.contains("Hello"));
}

#[wasm_bindgen_test]
fn test_parse_subtitles_unknown_format() {
  let result = parse_subtitles("garbage");
  assert!(!result.is_ok());
  assert_eq!(result.subtitle_count, 0);
}

#[wasm_bindgen_test]
fn test_detect_srt() {
  let srt = "1\n00:00:01,000 --> 00:00:03,500\nHello\n\n";
  let fmt = detect(srt);
  assert_eq!(fmt, "Srt");
}

#[wasm_bindgen_test]
fn test_detect_unknown() {
  assert_eq!(detect("garbage"), "unknown");
}

#[wasm_bindgen_test]
fn test_convert_format_srt_to_vtt() {
  let srt = "1\n00:00:01,000 --> 00:00:03,500\nHello\n\n";
  let result = convert_format(srt, "vtt");
  assert!(result.is_ok());
  assert!(result.output.contains("WEBVTT"));
}

#[wasm_bindgen_test]
fn test_validate_and_info() {
  let srt = "1\n00:00:01,000 --> 00:00:03,500\nHello\n\n";
  let v = validate_subtitles(srt);
  assert!(!v.is_null());
  let info = get_info(srt);
  assert!(!info.is_null());
}
```

**注意**：`#[wasm_bindgen_test]` 要求 `Cargo.toml` 加 `wasm-bindgen-test` 到 dev-dependencies。

### 2.2 W2+CI1+CI2: CI 改造

`.github/workflows/rust.yml` 加 3 个新 job：

```yaml
# wasm — 编译 + 运行 wasm-pack test（Node.js）
wasm:
  runs-on: ubuntu-latest
  steps:
    - uses: actions/checkout@v4
    - uses: dtolnay/rust-toolchain@stable
    - run: rustup target add wasm32-unknown-unknown
    - run: cargo build --target wasm32-unknown-unknown
    - run: curl https://rustwasm.github.io/wasm-pack/installer/init.sh -sSf | sh
    - run: wasm-pack test --node

# msrv — 用 1.85 toolchain 验证
msrv:
  runs-on: ubuntu-latest
  steps:
    - uses: actions/checkout@v4
    - uses: dtolnay/rust-toolchain@1.85
    - run: cargo build --verbose
    - run: cargo test --verbose

# clippy-matrix — 多个 feature 组合都跑 clippy
clippy-matrix:
  strategy:
    matrix:
      features:
        - ""
        - "--no-default-features --features srt"
        - "--features ttml"
  runs-on: ubuntu-latest
  steps:
    - uses: actions/checkout@v4
    - uses: dtolnay/rust-toolchain@stable
      with:
        components: clippy
    - run: cargo clippy --all-targets ${{ matrix.features }} -- -D warnings
```

**重要**：`wasm-pack test --node` 需要 GitHub Actions runner 上有 Node.js（ubuntu-latest 默认就有）。

### 2.3 B1: Benches CI（nightly，不阻塞）

```yaml
bench:
  runs-on: ubuntu-latest
  if: github.event_name == 'schedule' || contains(github.event.head_commit.message, '[bench]')
  steps:
    - uses: actions/checkout@v4
    - uses: dtolnay/rust-toolchain@nightly
    - run: cargo bench --no-run  # 编译但不运行（避免超时）
```

### 2.4 T1: Cross-format 稀疏矩阵

`tests/cross_format_matrix.rs` — 参数化测试，每格式至少 3 个出入向：

| 格式 | 出向（转换目标） | 入向（来源格式） |
|------|----------------|----------------|
| SRT | VTT, ASS, MicroDVD | VTT, ASS, MicroDVD |
| VTT | SRT, ASS, SBV | SRT, ASS |
| ASS | SRT, VTT | SRT, VTT, SSA |
| MicroDVD | SRT, SubViewer | SRT |
| SubViewer | SRT, MicroDVD | SRT |
| TTML | SRT, VTT | SRT |
| SBV | SRT, VTT | SRT |
| LRC | SRT | SRT |
| SAMI | SRT | SRT |
| MPL2 | SRT | SRT |
| SCC | SRT | SRT |
| EBU STL | — | SRT |

每个转换对验证：`parse(A) → to_string_format(B) → parse_as(B) → subtitle count 一致`。

### 2.5 T2: Chardetng 回退

`src/encoding.rs` 加 3 个 fixture 测试：

```rust
#[test]
fn test_decode_shift_jis() {
  // "こんにちは" in Shift_JIS
  let data = &[0x82, 0xB1, 0x82, 0xF1, 0x82, 0xC9, 0x82, 0xBF, 0x82, 0xCD];
  let result = decode_to_string(data).unwrap();
  assert_eq!(result, "こんにちは");
}
// 再加 GBK 和 Big5 fixture
```

### 2.6 T3: Proptest 扩展

- `arb_subtitle`: 文本加 Unicode（CJK/emoji/RTL）+ 换行 + HTML tags + ASS tags
- 加 ASS round-trip 属性测试
- 加 `shift_all` idempotency：`shift(-x) ∘ shift(x) ≈ id`（取模 clamping）
- 加 `sort` idempotency：`sort(sort(x)) == sort(x)`
- 加 `merge_adjacent` monotonicity：不增 cue 数 + 保留串联 text

### 2.7 T4: 错误路径

```rust
// src/ttml.rs #[cfg(test)]
#[test]
fn test_malformed_xml_triggers_error() {
  let result = parse_content("<tt><body><div><p>no closing tags");
  assert!(result.is_err());
}

// src/mpl2.rs #[cfg(test)]
#[test]
fn test_non_numeric_frame_returns_error() {
  let result = parse_content("{abc}{def}Hello\n");
  assert!(result.is_err());
}
```

## 5. 风险表

| 风险 | 概率 | 缓解 |
|------|------|------|
| wasm-pack 安装失败（CI 网络） | 中 | 用 curl 官方 installer，带 retry |
| wasm-bindgen-test 与某依赖冲突 | 低 | 先 `wasm-pack build` 通过再开 test |
| proptest 扩展导致测试时间暴增 | 中 | 限制 `#[proptest(cases = 100)]`，默认 256 |
| cross-format 矩阵编译慢 | 低 | 参数化测试，仍是单 target |
| chardetng fixture 字节选错（编码不匹配） | 中 | 用 Python 验证 `\u3053` → Shift_JIS → bytes |

## 6. CHANGELOG 草稿

```markdown
## [2.3.0] - YYYY-MM-DD

### Added

- **WASM tests**: 6 `#[wasm_bindgen_test]` tests covering `parse_subtitles`,
  `convert_format`, `validate_subtitles`, `detect`, `get_info`, and
  `normalize_text` in the browser target.
- **WASM CI job**: `wasm-pack test --node` runs on every push.
- **MSRV CI job**: verifies build + test pass on Rust 1.85.
- **Clippy feature matrix**: now runs across default, `--no-default-features
  --features srt`, and `--features ttml` to catch `#[cfg]`-gated issues.
- **Cross-format conversion matrix** (30 pairs): every format has at least
  3 incoming/outgoing conversion paths verified (parse → serialize → reparse).
- **Chardetng fallback tests**: Shift_JIS, GBK, and Big5 fixtures for
  `encoding::decode_to_string`.
- **Proptest expansion**: `arb_subtitle` now generates Unicode, newlines,
  and HTML/ASS tags; idempotency properties for shift, sort, and
  merge_adjacent.
- **Error-path tests**: malformed XML (TTML) and non-numeric frames
  (MPL2/MicroDVD) trigger typed `SubtitleError` variants.
- **Nightly bench CI job**: compiles criterion benchmarks (run on
  `[bench]` commit message or schedule).
```
