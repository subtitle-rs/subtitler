> 状态: Approved (2026-07-18)
> 范围: 5 项非 breaking 的 API 拉齐（A+B+C+E+I）
> 版本: 2.1.0 → 2.2.0 (minor，含 TTML deprecation)
> 路线图参考: `docs/superpowers/specs/2026-07-18-post-2.0-roadmap-design.md` §2.3

## 1. 范围

### 1.1 做

5 项纯加法/小修：

1. **A. 11 格式补 `generate(subs, path, policy)`** — 默认参数版（header=None / fps=DEFAULT_FPS / drop_frame=false）。
2. **B. 7 格式补 `parse_stream(content)` 工厂** — 每个 1 行 wrapper，签名统一为返回具体 Stream 类型（如 `MicroDvdStream<'a>`），与 srt/vtt 风格一致。
3. **C. TTML 新增 `write_stream_async<W: AsyncWrite + Unpin>`** — 旧 `write_stream` 加 `#[deprecated]`。内部用 in-memory buffer 桥接 sync quick-xml。
4. **E. TTML `to_string` 实现 header 保留** — `_header` 改名 `header`，实际写入 `<head>` 块。核查 parse 侧，若未提取则补。
5. **I. sami/mpl2/scc 补 `write_stream<W: AsyncWrite + Unpin>`** — 套模板（内部调 to_string + write_all）。

附加：抽 `write_with_policy` 共享 helper（Task 1），让 srt/vtt 也 DRY（Task 9）。

### 1.2 不做（推到 3.0 或不改）

- ❌ **D. 返回类型统一** — 7 格式 `parse_content` breaking。3.0 配 deprecation 周期做。
- ❌ **G. ass/ssa `to_string` 三参数重设计** — 需独立设计（builder 模式？），3.0 做。
- ❌ **J. ass/ttml StreamingParser impl** — 流式解析 XML/ASS 需重新设计状态机，3.0 做。
- ❌ **H. ebu_stl 二进制特殊性** — 不改，二进制本质。
- ❌ **F. MicroDVD `fps` 贯穿** — 不改，功能必需。

### 1.3 版本号

`2.1.0 → 2.2.0`。**minor**：纯加法 + TTML 的 `#[deprecated]`（deprecation 不算 breaking，但配 MIGRATION 说明）。**无任何函数签名删除或语义改变**。

## 2. 技术方案

### 2.1 Fix 1 (A): 11 格式补 `generate`

`generate` 是 `to_string + tokio::fs::write + WritePolicy` 的组合。模板（以 mpl2 为例）：

```rust
/// Write subtitles to a file in MPL2 format.
///
/// `policy` controls overwrite behavior (None = default Overwrite).
/// For format-specific options (fps), call `to_string` directly and
/// write the result with `tokio::fs::write`.
#[cfg(not(target_arch = "wasm32"))]
pub async fn generate(
  subtitles: &[Subtitle],
  file_path: impl AsRef<std::path::Path>,
  policy: Option<crate::model::WritePolicy>,
) -> AnyResult<()> {
  let content = to_string(subtitles, None);  // 默认参数
  crate::model::builder::write_with_policy(file_path, content.as_bytes(), policy).await
}
```

**共享 helper**（Task 1）：把 WritePolicy 处理逻辑抽到 `src/model/builder.rs`（已有 `WritePolicy` 类型在那）：

```rust
// src/model/builder.rs (or src/io.rs)
#[cfg(not(target_arch = "wasm32"))]
pub async fn write_with_policy(
  path: impl AsRef<std::path::Path>,
  bytes: &[u8],
  policy: Option<WritePolicy>,
) -> AnyResult<()> {
  use tokio::io::AsyncWriteExt;
  let path = path.as_ref();
  let policy = policy.unwrap_or(WritePolicy::Overwrite);
  match policy {
    WritePolicy::Overwrite => {
      let mut f = tokio::fs::OpenOptions::new()
        .write(true).truncate(true).create(true).open(path).await?;
      f.write_all(bytes).await?;
      f.flush().await?;
    }
    WritePolicy::RefuseIfExists if path.exists() => {
      return Err(anyhow::anyhow!(crate::error::SubtitleError::FileExists { path: path.to_path_buf() }.into()));
    }
    WritePolicy::RefuseIfExists => {
      let mut f = tokio::fs::File::create(path).await?;
      f.write_all(bytes).await?;
      f.flush().await?;
    }
    WritePolicy::Append => {
      let mut f = tokio::fs::OpenOptions::new()
        .append(true).create(true).open(path).await?;
      f.write_all(bytes).await?;
      f.flush().await?;
    }
  }
  Ok(())
}
```

实施时先读现有 srt/vtt 的 generate（`grep -A 30 "pub async fn generate" src/srt.rs src/vtt.rs`）作为模板，抽取共享逻辑。

**11 个格式各自的默认参数**：
- ass/ssa: `to_string(&HashMap::new(), &[AssStyle::default_style()], subs)` — 用空 info + 默认样式。
- microdvd: `to_string(subs, None)` — None → DEFAULT_FPS。
- subviewer/sami: `to_string(subs, None)` — header=None。
- ttml: `to_string(subs, None)` — header=None（联动 Fix E）。
- sbv/lrc: `to_string(subs)` — 无额外参数。
- mpl2: `to_string(subs, None)` — fps=None → DEFAULT_FPS。
- scc: `to_string(subs, false)` — drop_frame=false。
- ebu_stl: 特殊 — `to_string` 返回 `Vec<u8>`，generate 写字节。helper 接受 `&[u8]`，直接传。

### 2.2 Fix 2 (B): 7 格式补 `parse_stream(content)` 工厂

每个格式加 1 行 wrapper：

```rust
// src/microdvd.rs
pub fn parse_stream<'a>(content: &'a str) -> MicroDvdStream<'a> {
  MicroDvdStream::new(content, None)   // None = 用 DEFAULT_FPS
}

// src/subviewer.rs
pub fn parse_stream<'a>(content: &'a str) -> SubViewerStream<'a> {
  SubViewerStream::new(content)
}

// src/sbv.rs / lrc.rs / sami.rs 同理
// src/mpl2.rs: Mpl2Stream::new(content, None)
// src/scc.rs: SccStream::new(content)
```

**返回具体类型**（如 `MicroDvdStream<'a>`）而非 `impl Trait`，与 srt/vtt 现有风格一致（它们也返回 `SrtStream<'a>` / `VttStream<'a>`）。这样用户能拿到具体类型的额外方法。

### 2.3 Fix 3 (C): TTML 新增 `write_stream_async`

```rust
// src/ttml.rs

/// Asynchronous streaming write (replaces write_stream).
///
/// Uses an internal in-memory buffer to bridge quick-xml's sync Writer
/// to tokio's AsyncWrite. TTML output is typically small enough that a
/// single buffer is fine; for true streaming with very large inputs,
/// a chunked approach would be needed (deferred to 3.0 XML rewrite).
#[cfg(not(target_arch = "wasm32"))]
pub async fn write_stream_async<W>(subtitles: &[Subtitle], writer: &mut W) -> AnyResult<()>
where
  W: tokio::io::AsyncWrite + Unpin + Send,
{
  use tokio::io::AsyncWriteExt;
  let mut buf = Vec::new();
  write_stream(subtitles, &mut buf)?;   // 调现有同步版本（未 deprecated 的内部调用）
  writer.write_all(&buf).await?;
  writer.flush().await?;
  Ok(())
}

/// Deprecated synchronous write. Use `write_stream_async` for new code.
#[deprecated(since = "2.2.0", note = "use write_stream_async instead")]
pub fn write_stream<W: std::io::Write>(subtitles: &[Subtitle], writer: &mut W) -> AnyResult<()> {
  // 现有实现不变
}
```

**关键设计**：用 `in-memory buffer + write_all` 桥接。注释明确说明这是权衡（大文件场景推 3.0）。

### 2.4 Fix 4 (E): TTML 实现 header 保留

修改 `to_string` 让 header（若有）注入 `<head>` 块。用 quick-xml 而非手动拼接，避免转义 bug：

```rust
pub fn to_string(subtitles: &[Subtitle], header: Option<&str>) -> String {
  // ... 用 quick-xml 写
  if let Some(h) = header.filter(|s| !s.is_empty()) {
    // 写入 <head> 块，h 作为 raw text（调用方负责合法 XML）
    // 或解析 h 后写（更安全，但复杂）—— 2.2 用 raw text + 注释说明
  }
}
```

**关键步骤**：实施时第一步核查 parse 侧 —— 检查 `parse_content` 是否填充 `SubtitleFile::Ttml { header, .. }`。如果没填，2.2 顺便补 parse 侧 header 提取。如果太复杂（需要解析 XML 树），只做 write 侧 + CHANGELOG 注明 "parse 侧 header 提取推到 3.0"。

参数名 `_header` → `header`（去掉下划线前缀，表示参数已生效）。

### 2.5 Fix 5 (I): sami/mpl2/scc 补 `write_stream`

每个格式加一个 async write_stream，内部调 `to_string` + `write_all`：

```rust
// src/sami.rs (mpl2/scc 同理)
#[cfg(not(target_arch = "wasm32"))]
pub async fn write_stream<W>(subtitles: &[Subtitle], writer: &mut W) -> AnyResult<()>
where
  W: tokio::io::AsyncWrite + Unpin,
{
  use tokio::io::AsyncWriteExt;
  let content = to_string(subtitles, None);  // header=None
  writer.write_all(content.as_bytes()).await?;
  writer.flush().await?;
  Ok(())
}

// src/mpl2.rs: to_string(subs, None)  // fps=None → DEFAULT_FPS
// src/scc.rs: to_string(subs, false)  // drop_frame=false
```

## 3. 实施顺序

每步独立 commit，串行（AGENTS.md §3.11：不并行 implementation）：

| # | 任务 | 文件 | 风险 | 依赖 |
|---|------|------|------|------|
| **1** | 抽 `write_with_policy` 共享 helper | `model/builder.rs` | 低（DRY 重构准备） | 无 |
| **2** | A1: ass/ssa 补 generate（用默认 info+style） | `ass.rs` | 低 | 1 |
| **3** | A2: microdvd/subviewer/sbv/lrc/mpl2/scc 补 generate | 6 文件 | 低（机械套模板） | 1 |
| **4** | A3: ttml/ebu_stl 补 generate（含 header/binary 特殊） | `ttml.rs`, `ebu_stl.rs` | 中（header 联动 E） | 1 |
| **5** | B: 7 格式补 `parse_stream(content)` 工厂 | 7 文件 | 极低（每个 1 行 wrapper） | 无 |
| **6** | I: sami/mpl2/scc 补 `write_stream`（async） | 3 文件 | 低（套模板） | 无 |
| **7** | E: TTML 实现 header 保留（含 parse 侧核查） | `ttml.rs` | 中（需核实 parse 是否提 header） | 无 |
| **8** | C: TTML 新增 `write_stream_async` + deprecate 旧版 | `ttml.rs` | 中（async bridge） | 无 |
| **9** | DRY srt/vtt：让它们也调共享 helper | `srt.rs`, `vtt.rs` | 低 | 1 |
| **10** | 13 格式 API 一致性测试 `tests/api_surface.rs` | 新文件 | 低 | 1-9 |
| **11** | 版本 bump + CHANGELOG + MIGRATION + CODE_WIKI | 文档 | 极低 | 1-10 |

**排序逻辑**：
- Task 1 先做 helper，后续所有 generate 都依赖它。先 DRY 再加新。
- Task 2-4 是 generate 的三批（简单 → 复杂）。ass/ssa 单独一批因为默认 info/styles 要思考。ebu_stl/ttml 单独一批因为 binary/header 特殊。
- Task 5-6 是纯加法（parse_stream + write_stream），独立无依赖。
- Task 7-8 是 TTML 的两项，C 放 E 后因为 deprecation 要最后（让 E 的 to_string 改动先稳定）。
- Task 9 DRY srt/vtt 放最后，因为这是重构已有代码，风险最高（虽然只是内部 DRY）。
- Task 10 API 一致性测试锁住拉齐结果。
- Task 11 文档（严格按 AGENTS.md §6.7 / §7.2：CODE_WIKI 是发版 gate）。

## 5. 风险表

| 风险 | 概率 | 影响 | 缓解 |
|------|------|------|------|
| ass 默认 `info`/`styles` 导致输出 ASS 文件无法被 libass 解析 | 中 | 中 | 用 `AssStyle::default_style()`（已存在）+ 空 info；测试 round-trip 可 parse 回 |

## 6. CHANGELOG 草稿

```markdown
## [2.2.0] - YYYY-MM-DD

### Added

- **`generate(subs, path, policy)` for all 13 formats** — previously only
  SRT and VTT had this file-write helper. The other 11 formats (ASS, SSA,
  MicroDVD, SubViewer, TTML, SBV, LRC, SAMI, MPL2, SCC, EBU STL) now
  expose the same signature with format-appropriate defaults. For
  format-specific options (fps, drop_frame, header), call `to_string`
  directly and write with `tokio::fs::write`.
- **`parse_stream(content)` factory for 7 more formats** — MicroDVD,
  SubViewer, SBV, LRC, SAMI, MPL2, SCC now expose a public
  `parse_stream` constructor matching the SRT/VTT pattern (previously
  users had to call `XxxStream::new` directly).
- **`write_stream` for SAMI, MPL2, SCC** — these three formats had
  `StreamingParser` reads but no streaming write; now symmetric.
- **TTML `write_stream_async`** — async alternative to the deprecated
  sync `write_stream`. Bridges quick-xml's sync Writer via an in-memory
  buffer.
- **TTML header preservation** — `to_string(subs, header)` now actually
  writes the header into a `<head>` block instead of silently dropping
  it. Parameter renamed `_header` → `header`.
- **Shared `write_with_policy` helper** in `model::builder` — DRYs the
  WritePolicy logic that SRT/VTT were duplicating; all 13 formats now
  use it.

### Deprecated

- `ttml::write_stream` (sync) — replaced by `write_stream_async`.
  Scheduled removal in 3.0.

### See also

- `MIGRATION.md` "2.1 → 2.2" section: TTML `write_stream` deprecation.
```

### 6.1 MIGRATION 增补草稿

```markdown
## 2.1 → 2.2

### Additions (non-breaking)

- 11 more formats now expose `generate(subs, path, policy)`. Existing
  SRT/VTT users see no change.
- 7 more formats now expose `parse_stream(content)`.
- SAMI/MPL2/SCC now have `write_stream`.

### Deprecation

- `ttml::write_stream` (sync) is deprecated. Migrate to
  `ttml::write_stream_async`:
  ```rust
  // Before
  ttml::write_stream(&subs, &mut file)?;
  // After
  ttml::write_stream_async(&subs, &mut file).await?;
  ```
  The sync version remains until 3.0.

### Behavior change (TTML)

- `ttml::to_string(subs, header)` no longer ignores `header`. If you
  were passing `Some("...")` expecting it to be dropped, the output
  will now contain a `<head>` block.
```
