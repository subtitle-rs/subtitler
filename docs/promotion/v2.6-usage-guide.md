# 用 Rust 解析 15 种字幕格式：subtitler 库实战指南

> 10,718 行 Rust 代码，15 种字幕格式，344 个测试，零 clippy 警告。
> 这篇文章用实际代码示例展示 subtitler v2.6 能做什么。

## 30 秒上手

```toml
[dependencies]
subtitler = "2.6"
```

```rust
use subtitler::{parse_bytes, SubtitleFormat};

fn main() -> anyhow::Result<()> {
    let data = std::fs::read("movie.srt")?;
    let file = parse_bytes(&data)?;  // 自动检测格式

    println!("{} subtitles, format: {:?}",
        file.subtitles().len(),
        file.format());

    Ok(())
}
```

## 支持的 15 种格式

| 领域 | 格式 | 特点 |
|------|------|------|
| Web | SRT, VTT, TTML, DFXP | 主流 Web 字幕 |
| 视频编辑 | ASS, SSA | 富样式 + 动画 |
| DVD | MicroDVD, SubViewer | 帧时间码 |
| 广播 | SCC (CEA-608), EBU STL | NTSC drop-frame / 二进制 |
| YouTube | SBV | 两行格式 |
| 卡拉OK | LRC | 多时间戳歌词 |
| 微软 | SAMI | HTML 类多语言 |
| 东欧 | MPL2 | 帧格式 |
| AI | Whisper JSON | OpenAI Whisper 转录桥接 |

每种格式都是独立 feature flag，精简构建只需你要的格式：

```toml
[dependencies]
subtitler = { version = "2.6", default-features = false, features = ["srt", "vtt"] }
```

## 格式转换：一行搞定

```rust
use subtitler::{parse_bytes, SubtitleFormat, model::Format};

let data = std::fs::read("input.srt")?;
let file = parse_bytes(&data)?;

// 转 VTT
let vtt = file.to_string_with_format(&Format::Vtt);
std::fs::write("output.vtt", vtt)?;

// 转 ASS
let ass = file.to_string_with_format(&Format::Ass);
std::fs::write("output.ass", ass)?;
```

15×15 = 225 种转换路径，全部通过 parse → serialize → reparse 测试验证。

## Pipeline DSL：声明式字幕处理

这是 subtitler 独有的特性——用 JSON 配置文件描述多步操作链：

```json
{
  "operations": [
    { "op": "Sort" },
    { "op": "Shift", "offset_ms": 500 },
    { "op": "RemoveDuplicates" },
    { "op": "SplitLong", "max_chars": 42 },
    { "op": "MergeAdjacent", "max_gap_ms": 200 }
  ]
}
```

```bash
subtitler pipeline input.srt output.vtt --config ops.json
```

或代码方式：

```rust
use subtitler::pipeline::{Pipeline, SubtitleBuilder};

let file = SubtitleBuilder::from(file)
    .sort()
    .shift(500)
    .remove_duplicates()
    .split_long(42)
    .merge_adjacent(200)
    .build();
```

10 种 PipelineOp：Sort, Shift, MergeAdjacent, SplitLong, TransformFps,
RemoveOverlaps, EnforceMinDuration, EnforceMaxDuration, AutoExtendCps,
FilterEmpty, RemoveDuplicates。

## 验证：不只是解析

```rust
use subtitler::SubtitleFormat;

let file = parse_bytes(&data)?;
let issues = file.validate_extended(
    42,    // max_chars per subtitle
    5000,  // max_gap_ms
    25.0,  // max_cps (characters per second)
);

for issue in &issues {
    println!("{}", issue.description());
}
// → "subtitle 3 has CPS 31.2 (max 25.0)"
// → "gap between #5 and #6 is 8000ms (max 5000ms)"
```

## Whisper JSON → SRT 管道

v2.4 新增的 Whisper JSON 格式让 subtitler 成为 AI 转录的桥梁：

```rust
// Whisper 输出 → SRT
let whisper = std::fs::read_to_string("transcript.json")?;
let file = subtitler::whisper::parse_content(&whisper)?;
let srt = file.to_string_with_format(&Format::Srt);
std::fs::write("output.srt", srt)?;
```

```bash
# CLI 方式
subtitler convert transcript.json output.srt
```

## 编码自动检测

UTF-8 / UTF-16 (BE/LE) / GBK / Shift_JIS / Big5 —— 全部自动处理：

```rust
// GBK 编码的 SRT 文件？没问题。
let data = std::fs::read("chinese.srt")?;  // GBK 字节
let file = subtitler::parse_bytes(&data)?;  // 自动解码
```

## 流式解析：大文件友好

```rust
use subtitler::model::StreamingParser;

let mut parser = subtitler::srt::parse_stream(&content);
while let Some(result) = parser.next() {
    let sub = result?;
    println!("{} --> {}: {}", sub.start, sub.end, sub.text);
}
```

9 种格式支持流式解析，不一次性分配全部 `Vec<Subtitle>`。

## CLI 工具

```bash
subtitler parse movie.srt              # 解析并展示
subtitler convert input.srt output.vtt # 格式转换
subtitler validate movie.srt --json    # 验证 + JSON
subtitler detect unknown.sub           # 格式检测
subtitler info movie.srt               # 文件统计
subtitler edit in.srt --sort --shift 500 --output out.srt
subtitler normalize in.srt --all --output clean.srt
subtitler quality movie.srt --json     # 质量报告
subtitler pipeline in.srt out.vtt --config ops.json
```

## 为什么选 subtitler？

1. **15 格式**全量支持，远超同类工具
2. **零拷贝解析**——解析路径无临时分配
3. **Pipeline DSL**——声明式多步操作，JSON 可序列化
4. **344 测试**，clippy 零警告，MSRV 1.85
5. **WASM 就绪**——浏览器端直接用
6. **Whisper JSON 桥接**——AI 转录工作流
7. **Feature flags**——精简构建只编你要的格式
