# Whisper + Rust：用 subtitler 搭建 AI 转录字幕管道

> OpenAI Whisper 把语音转成 JSON 格式的转录本。subtitler v2.4 新增的 Whisper JSON 格式让它成为"AI 转录 → 任何字幕格式"的桥梁。

## 场景

你用 Whisper 做了一段视频的语音识别，拿到 `transcript.json`。你需要：

1. 转成 SRT 给播放器用
2. 转成 VTT 给网页用
3. 转成 ASS 给视频编辑软件用
4. 统一时间码偏移、去重、拆分长行

传统做法：手写 Python 脚本逐个处理。subtitler 做法：一条命令。

## Whisper JSON 长什么样

```json
{
  "text": "Hello world. Goodbye world.",
  "segments": [
    {
      "id": 0,
      "start": 0.0,
      "end": 2.5,
      "text": "Hello world.",
      "seek": 0,
      "tokens": [50364, 2425, 1000, 13],
      "temperature": 0.0,
      "avg_logprob": -0.3,
      "compression_ratio": 1.2,
      "no_speech_prob": 0.1
    },
    {
      "id": 1,
      "start": 3.0,
      "end": 5.5,
      "text": "Goodbye world.",
      "seek": 0,
      "tokens": [50364, 3513, 1000, 13],
      "temperature": 0.0,
      "avg_logprob": -0.3,
      "compression_ratio": 1.2,
      "no_speech_prob": 0.1
    }
  ]
}
```

`start`/`end` 是秒（f64），`text` 是文本。subtitler 把秒转成毫秒（×1000 round），每条 segment 变成一个 `Subtitle`。

## CLI：一条命令搞定

```bash
# Whisper JSON → SRT
subtitler convert transcript.json output.srt

# Whisper JSON → VTT
subtitler convert transcript.json output.vtt

# Whisper JSON → ASS
subtitler convert transcript.json output.ass
```

格式自动检测——subtitler 看到 `"segments"` + `"start"` + `"end"` 就知道是 Whisper JSON。

## 代码：管道式处理

```rust
use subtitler::SubtitleFormat;
use subtitler::model::Format;

// 解析 Whisper JSON
let json = std::fs::read_to_string("transcript.json")?;
let file = subtitler::whisper::parse_content(&json)?;

// 一步到位：偏移 + 去重 + 拆长行 + 转 SRT
use subtitler::pipeline::SubtitleBuilder;
let processed = SubtitleBuilder::from(file)
    .shift(500)              // 延迟 500ms 对齐音轨
    .remove_duplicates()     // 去重
    .split_long(42)          // 每行最多 42 字符
    .enforce_min_duration(1000)  // 最短 1 秒
    .build();

let srt = processed.to_string_with_format(&Format::Srt);
std::fs::write("output.srt", srt)?;
```

## Pipeline 配置文件

同样的操作链可以用 JSON 描述，方便复用：

```json
{
  "operations": [
    { "op": "Shift", "offset_ms": 500 },
    { "op": "RemoveDuplicates" },
    { "op": "SplitLong", "max_chars": 42 },
    { "op": "EnforceMinDuration", "min_ms": 1000 },
    { "op": "Sort" }
  ]
}
```

```bash
subtitler pipeline transcript.json output.srt --config pipeline.json
```

## 批量处理

Whisper 逐段处理长视频时可能产生多个 JSON。配合 subtitler 的 `concatenate`：

```rust
// 合并多个 Whisper 片段
let mut combined = subtitler::whisper::parse_content(&part1)?;
let part2 = subtitler::whisper::parse_content(&part2)?;
combined.concatenate(&part2, 1000);  // 1 秒间隔

// 导出
let srt = combined.to_string_with_format(&Format::Srt);
```

## 为什么不直接用 Python？

可以。但 Rust 版本的优势：

1. **15 种输出格式**——Python 库通常只支持 3-4 种
2. **零拷贝解析**——10 万行 JSON 比 Python 快 10 倍+
3. **编译到 WASM**——可以在浏览器端处理（上传 JSON，下载 SRT）
4. **Pipeline 可序列化**——配置文件描述操作链，不写代码
5. **Feature flags**——只编译你要的格式，二进制约 2MB

## 完整工作流

```
音频 → Whisper (Python/CUDA) → transcript.json
                                    ↓
                              subtitler convert
                                    ↓
                    SRT / VTT / ASS / TTML / SCC ...
```

Whisper 负责语音识别，subtitler 负责格式转换 + 后处理。各做各擅长的。
