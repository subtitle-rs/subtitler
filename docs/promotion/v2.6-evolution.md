# subtitler v2.6：一个 Rust 字幕库的进化史

> 从 v1.0 的 9 格式 216 测试，到 v2.6 的 15 格式 344 测试。
> 这不是简单的"加格式"——是 8 个版本迭代里，每一轮都在还债、拉齐、夯实。

## 起点：v2.0 的广度优先

v2.0 一次性交付了 13 种字幕格式 + WASM 支持 + Pipeline DSL + StreamingParser。这是"广度优先"策略的典型产物——功能覆盖面够了，但深度跟不上：

- SCC 的 drop-frame 时间码算法没实现，NTSC 长视频每小时漂移 3.6 秒
- EBU STL 的二进制 round-trip 是坏的（`ms / 40` 魔法数）
- UTF-16 解码不剥 BOM
- 13 种格式的公共 API 表面碎片化（只有 srt/vtt 有 `generate`，只有 srt/vtt 暴露 `parse_stream`）

## v2.1：正确性还债

6 个 P1 正确性 bug，每个配套 regression 测试：

**SCC drop-frame** 是最难的一个。SMPTE 12M-1-2014 标准算法不复杂（每分钟丢 2 帧，每 10 分钟不丢），但**测试预期值我手算错了两次**。最终用 Python 模拟交叉验证才拿到正确值：

```python
# drop-frame 01:00:00;00 = 3600000ms 正好 1 小整（drop-frame 的设计目的）
# non-drop 01:00:00:00 = 3603604ms（多 108 帧 ≈ 3.6s）
```

这个教训直接写进了 AGENTS.md §3.12：**TDD 数值预期必须独立交叉验证，不能纸面推导**。

## v2.2：API 拉齐

11 个格式补齐 `generate(subs, path, policy)`，7 个格式补齐 `parse_stream(content)` 工厂函数，3 个格式补齐 `write_stream`。TTML 的同步 `write_stream` 加了 `write_stream_async` + `#[deprecated]` 过渡。

关键产出：**编译期 API 一致性测试**（`tests/api_surface.rs`）——每个格式模块必须暴露 `parse_content`/`parse_bytes`/`parse_stream`/`to_string`/`detect_format`。如果未来哪个格式漏掉某个 API，CI 立刻编译失败。

## v2.3：测试/CI 夯实

WASM 6 个 `#[wasm_bindgen_test]`、cross-format 转换矩阵 23 对、proptest 扩展到 Unicode CJK、chardetng 编码回退测试（Shift_JIS + GBK fixture）、CI 加了 MSRV job + clippy feature 矩阵。

## v2.4：gap analysis 收编

对比 editingtools.io 后发现：DFXP 格式和 Whisper JSON 是低投入高回报。DFXP 与 TTML 共享 90% 代码，Whisper JSON 让 subtitler 成为"Whisper 输出 → 任何字幕格式"的管道中间件。

同时加了 `RemoveDuplicates` PipelineOp 和 4 个 normalize 函数（语言过滤、短行合并、换行替换）。

## v2.4.1：外部测试修复

用 editingtools.io 的真实 demo 文件跑了一遍 CLI 端到端测试，发现 **SCC 文本解码完全坏了**（v2.0 起所有 SCC 字幕文本静默丢失）。根因是 `decode_scc_hex` 把 4 位 hex token 当 u16 检查 ASCII 范围，但 SCC hex 是 2-byte pair（如 `5468` = 'T' + 'h'）。

这种 bug 只有用真实文件测试才能发现——单元测试用的是精心构造的输入。

## v2.6：代码质量

- **错误类型统一**：7 个格式的 `parse_content` 从 `Result<_, SubtitleError>` 改为 `AnyResult`，还原 v2.0 设计意图
- **Magic number 常量化**：`MS_PER_HOUR`/`MS_PER_MINUTE`/`MS_PER_SECOND`/`DEFAULT_TAIL_MS`
- **MSRV 修复**：消除 Rust 1.86 才稳定的 `let`-chain 语法

## 教训

1. **广度优先的代价是深度债务**——功能覆盖面够了但正确性跟不上
2. **TDD 的测试输入必须真能触发 bug**——用精心构造的输入写测试等于没测
3. **数值预期必须独立验证**——纸面推导会错，Python 模拟不会
4. **CI 矩阵不是奢侈品**——MSRV + clippy feature 矩阵在最关键时刻抓住了 let-chain 不兼容

## 下一步

2.x 已进入稳定打磨期。未来方向包括更多格式支持、AI 翻译集成、跨语言绑定——但那是新的篇章。
